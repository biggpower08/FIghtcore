# Sprite Frame Locations

This file is generated by `npm.cmd run sprite-pack:list-frames -- --write-doc`.

Runtime priority: `manual-overrides` > `frames-cleaned` > `frames-pack` > `frames-reference` > `frames-semi-realistic` > raw fallback.

Manual edits belong in `public/sprites/manual-overrides/<character>/<animation>/0001.png`. Missing override frames fall back normally, one frame at a time.

Manual override workflow: copy only the bad transparent frame into `public/sprites/manual-overrides/<character>/<animation>/<frame>.png`, clean that single PNG by hand, keep the same filename, and leave every other frame out of the override folder. Runtime checks overrides first, then cleaned/generated/fallback frames without renumbering the animation.

Do not hand-edit generated runtime folders unless you are comfortable losing those edits during an importer run.

PowerShell helpers:

```powershell
npm.cmd run sprite-pack:list-frames
npm.cmd run sprite-pack:list-frames -- --write-doc
Get-ChildItem "public\sprites\manual-overrides" -Recurse -Filter *.png | Select-Object FullName
Get-ChildItem "public\sprites\frames-pack" -Recurse -Filter *.png | Select-Object FullName
```
